# PepsiStreamy 🥤🎶

**내 PC에서 나는 소리(브라우저에서 재생 중인 유튜브 등)를 디스코드 음성 채널로 실시간 방송**하는 작은 CLI 프로그램입니다.
디스코드 봇이 음성 채널에 들어가 내 컴퓨터 소리를 흘려보내고, **듣고 싶은 사람은 그 채널에 들어오기만** 하면 됩니다.

> 포트포워딩도, 빌리는 서버도 필요 없습니다. 봇이 디스코드로 *바깥으로* 접속(outbound)하기 때문입니다.

```
 [ 내 PC (Windows) ]
   브라우저 (유튜브 로그인·재생)
        │  스피커로 출력
        ▼
   WASAPI 루프백 캡처 ──► 20ms PCM 큐 ──► PepsiStreamy 봇 (Opus 인코딩)
                                               │ Discord로 outbound 접속
                                               ▼
                                        [ Discord 음성 채널 ]
                                               │
                                          듣는 사람들 🎧
```

## 왜 서버·포트포워딩이 필요 없나

외부에서 내 PC로 **접속해 들어오는(inbound)** 구조라면 포트포워딩/공인 IP가 필요합니다.
하지만 PepsiStreamy는 **디스코드 봇**을 씁니다. 봇은 디스코드 서버로 **바깥으로 나가는(outbound)** 연결만 만들고, 듣는 사람도 각자 디스코드로 접속합니다. 디스코드가 가운데에서 만나게 해주므로 **내 PC를 외부에 열 필요가 전혀 없습니다.**

또한 "로그인된 유튜브"는 **내 브라우저에서 재생**하고 PC가 그 소리를 잡아 보내므로, 구글 로그인 정보가 어디 외부 서버로 올라가는 일도 없습니다.

## 최소화·백그라운드 탭도 잡히나? → 네

이 프로그램은 Windows의 **WASAPI 루프백**으로 "스피커로 나가는 최종 믹스"를 캡처합니다.
이 캡처는 **창의 상태(최소화 / 백그라운드 / 포커스 여부)와 무관**합니다. 크롬·엣지 등은 백그라운드 탭이어도 **오디오 재생은 멈추지 않으므로**(타이머만 절전 처리), 탭을 최소화하거나 뒤로 보내도 **소리만 나면 그대로 방송**됩니다.

예외:
- 탭/앱을 **음소거**하면 당연히 안 잡힙니다(소리가 없으니까).
- 브라우저 출력을 **기본 스피커가 아닌 다른 장치**로 바꿔뒀다면, 그 장치를 `YTCAST_DEVICE`로 지정하세요. (`devices` 명령으로 이름 확인)

## 요구사항

- **Windows 10/11** (WASAPI 루프백 캡처 사용)
- **Python 3.9+** ([python.org](https://www.python.org/downloads/) 설치 시 "Add to PATH" 체크)
- 디스코드 계정 + 내가 봇을 추가할 수 있는 서버
- 의존성(자동 설치됨): `discord.py[voice]`, `soundcard`, `numpy`, `python-dotenv`
  - 음성 송출에 필요한 `PyNaCl`/`libopus`는 `discord.py[voice]`에 포함되어 자동 처리됩니다.

## 빠른 시작

```powershell
# 1) 코드 받기
git clone https://github.com/ssapgosu12/pepsistreamy.git
cd pepsistreamy

# 2) 설정 파일 만들기 (아래 '디스코드 봇 만들기'에서 토큰 발급)
Copy-Item .env.example .env
notepad .env          # DISCORD_TOKEN= 에 내 봇 토큰 붙여넣기

# 3) 런처 실행 — 첫 실행 시 .venv 가상환경 + 의존성을 자동 설치합니다
./run.ps1 doctor      # 환경 점검 (전부 OK + 토큰 OK 면 준비 끝)
./run.ps1 meter       # 유튜브 틀어놓고 막대가 움직이는지 확인
./run.ps1 run         # 봇 실행
```

> **PowerShell 실행정책에 막히면**: `powershell -ExecutionPolicy Bypass -File run.ps1 run`

## 디스코드 봇 만들기 (최초 1회)

1. **애플리케이션 생성**: <https://discord.com/developers/applications> → 우측 상단 **New Application** → 이름 입력(예: PepsiStreamy) → Create.
2. **봇 토큰 발급**: 좌측 메뉴 **Bot** → **Reset Token** → **Copy**.
   - 이 토큰은 **비밀번호**입니다. 누구에게도 공유하지 말고, `.env`에만 넣으세요.
   - **특권 인텐트(Privileged Gateway Intents)는 켤 필요 없습니다.** 이 봇은 슬래시 명령만 사용합니다.
3. **토큰 저장**: `.env.example`을 `.env`로 복사한 뒤 `DISCORD_TOKEN=` 뒤에 붙여넣기.
4. **서버에 초대**: 좌측 **OAuth2 → URL Generator** 에서
   - **SCOPES**: `bot`, `applications.commands` 체크
   - **BOT PERMISSIONS**: `View Channels`, `Connect`, `Speak` 체크
   - 맨 아래 생성된 URL을 브라우저에서 열고 → 내 서버 선택 → 승인.
5. (권장) **명령 즉시 표시**: `.env`의 `DISCORD_GUILD_ID=`에 서버 ID를 넣으면 슬래시 명령이 곧바로 보입니다.
   비워두면 전역 동기화라 명령이 보이기까지 최대 1시간 걸립니다.
   - 서버 ID: 디스코드 **설정 → 고급 → 개발자 모드** 켠 뒤, 서버 아이콘 **우클릭 → 서버 ID 복사**.

## 사용법

```powershell
./run.ps1 doctor      # 의존성/opus/토큰 점검
./run.ps1 devices     # 캡처 가능한 출력/입력 장치 목록
./run.ps1 meter       # 선택한 장치의 캡처 레벨을 실시간으로 확인 (--seconds 로 시간 조절)
./run.ps1 run         # 캡처 + 봇 실행 (Ctrl+C 로 종료)
```

봇을 실행한 뒤 **디스코드에서**:

1. 듣고 싶은 **음성 채널에 내가 먼저 입장**한다.
2. 채팅창에 **`/join`** → 봇이 그 채널에 들어와 방송을 시작.
3. 브라우저에서 **유튜브 재생** → 채널에 있는 모두에게 들립니다.
4. **`/status`** 로 상태 확인, **`/leave`** 로 종료.

| 명령 | 위치 | 설명 |
|------|------|------|
| `run.ps1 doctor` | CLI | 의존성·opus·토큰 점검 |
| `run.ps1 devices` | CLI | 출력(루프백)/입력 장치 목록 |
| `run.ps1 meter` | CLI | 캡처 레벨 실시간 측정 |
| `run.ps1 run` | CLI | 캡처 + 봇 실행 |
| `/join` | Discord | 내가 있는 음성 채널로 들어가 방송 시작 |
| `/leave` | Discord | 방송 종료 후 채널 나가기 |
| `/status` | Discord | 캡처/연결 상태 보기 |

> `run.ps1` 없이 직접 쓰려면: 가상환경 활성화 후 `python -m pepsistreamy <명령>`.

## 특정 "탭"만 방송하기

기본(루프백)은 **시스템 전체 믹스**라 그 탭 소리 + 윈도우 알림음·다른 앱 소리가 함께 나갑니다.
Windows에서 크롬은 **모든 탭의 오디오를 하나의 오디오 프로세스에서 믹스**하기 때문에, OS 차원에서 **단일 탭만** 떼어내는 것은 불가능합니다. 대신 다음 방법을 씁니다.

**방법 A — 다른 소리를 안 내기 (가장 간단)**
그 유튜브 탭만 소리를 내고, 윈도우 알림과 다른 앱은 음소거. 기본 루프백 그대로 사용.

**방법 B — 가상 케이블로 그 브라우저만 분리 (권장)**
1. 무료 [VB-CABLE](https://vb-audio.com/Cable/) 설치 → 재생장치 `CABLE Input`, 녹음장치 `CABLE Output` 이 생깁니다.
2. 방송할 탭을 **전용 브라우저/프로필**에 띄웁니다(단일 탭 분리를 위해. 예: 크롬 새 프로필, 또는 Edge/Firefox 전용 창).
3. 윈도우 **설정 → 시스템 → 소리 → 볼륨 믹서**에서 그 브라우저 앱의 **출력 장치**를 `CABLE Input` 으로 변경.
4. `.env` 에 `YTCAST_DEVICE=CABLE` 지정 → 그 케이블만 캡처(다른 시스템 소리 제외).
5. 본인도 같이 들으려면 `CABLE Output` 속성에서 **이 장치 듣기**를 켜 스피커로도 보냅니다.

이렇게 하면 그 브라우저 창을 **최소화/백그라운드로 둬도** 소리만 나면 계속 방송됩니다.

## 여러 사람과 함께 쓰기

핵심: **봇은 `run`을 돌리는 그 컴퓨터의 소리를 방송합니다.** 그래서 친구가 *자기* 컴퓨터 소리를 *자기* 서버로 방송하려면—

- ❌ **내 토큰을 공유하면 안 됩니다.** 토큰은 봇의 비밀번호이고, 공유해도 그건 "내 봇 1개가 내 PC 소리만" 내보내는 것이라 친구의 PC 소리는 못 보냅니다.
- ✅ **각자 자기 봇을 만듭니다.** 친구도 위 [디스코드 봇 만들기](#디스코드-봇-만들기-최초-1회)를 따라 **자기 토큰**을 만들고 → **자기 서버에 초대**하고 → **자기 PC에서 실행**하면 됩니다.

즉 공유하는 것은 **이 코드(레포)** 뿐입니다. 각자 `git clone` → `.env`에 자기 토큰 → `run.ps1 run`.
`.gitignore`가 `.env`와 `.venv`를 제외하므로 **토큰이 레포에 올라갈 일은 없습니다.**

## 접근 제어 (프라이버시)

누가 들을 수 있는지는 **그 음성 채널의 디스코드 권한**으로 정합니다.
나만/지정한 사람만 듣게 하려면 **비공개 음성 채널**을 만들고 그 채널의 "채널 보기/연결" 권한을 특정 역할·사용자로 제한하세요.

## 문제 해결

| 증상 | 해결 |
|------|------|
| `/join` 명령이 안 보임 | `.env`에 `DISCORD_GUILD_ID` 넣고 다시 `run` (전역 동기화는 최대 1시간) |
| 봇은 들어왔는데 소리가 안 남 | `./run.ps1 meter` 로 캡처 레벨 확인 → 0이면 장치 문제. 브라우저 출력 장치와 `YTCAST_DEVICE`가 같은지 확인. 탭 음소거 해제 |
| `run.ps1` 이 실행 안 됨 | `powershell -ExecutionPolicy Bypass -File run.ps1 run` |
| `doctor`에서 libopus가 X | 보통 `discord.py[voice]` 재설치로 해결. 가상환경 지우고 `run.ps1` 재실행 |
| 소리가 끊기거나 지연 큼 | 디스코드 음성 채널 비트레이트를 64~96kbps로 낮추기, CPU 여유 확인 |
| 토큰 오류로 종료 | `.env`의 `DISCORD_TOKEN` 값 확인(따옴표 없이), 개발자 포털에서 토큰 재발급 |
| 소리에 다른 앱·알림이 섞임 | [특정 탭만 방송하기](#특정-탭만-방송하기) 참고 |

## 동작 원리

1. **캡처** (`capture.py`): `soundcard`의 WASAPI 루프백으로 출력 장치 믹스를 48kHz·스테레오로 읽어, 20ms(=3840바이트 s16le) 프레임을 만들어 큐에 쌓습니다. 큐는 최신 우선(가득 차면 오래된 프레임을 버림)이라 지연이 무한정 늘지 않습니다.
2. **소스** (`source.py`): 디스코드 음성 플레이어가 20ms마다 한 프레임을 요청하면 큐에서 꺼내 줍니다. 버퍼가 비면 무음을 돌려줘 연결만 유지합니다.
3. **봇** (`bot.py`): 슬래시 명령(`/join` `/leave` `/status`)만 사용하므로 특권 인텐트가 필요 없습니다. `/join` 시 명령을 친 사람이 있는 음성 채널로 들어가 위 소스를 Opus로 인코딩해 송출합니다.

## 프로젝트 구조

```
pepsistreamy/
├─ pepsistreamy/
│  ├─ __init__.py
│  ├─ __main__.py     # python -m pepsistreamy 진입점
│  ├─ capture.py      # WASAPI 루프백 캡처 → PCM 프레임 큐
│  ├─ source.py       # 큐 → Discord AudioSource
│  ├─ bot.py          # 슬래시 명령 봇 (/join /leave /status)
│  └─ cli.py          # doctor / devices / meter / run
├─ run.ps1            # 가상환경+의존성 자동설치 런처
├─ requirements.txt
├─ .env.example       # 설정 템플릿 (.env 로 복사해 사용)
├─ .gitignore         # .env / .venv 제외
└─ README.md
```
